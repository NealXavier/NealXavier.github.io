Object类的equals方法

发表于 2017-08-18 | 分类于 Java基础

equals方法的源代码写法：

public boolean equals(Object obj) {return (this == obj);}

从以上源代码可以看出，默认情况比较的是地址值，因为比较地址值一般来说意义不大所以我们要重写该方法。改写成比较该类的所有属性是否相等的equals方法。

equals方法的改写

public boolean equals(Object obj) {
    //为了提高效率

    //源代码写法，对象都是同一个肯定相同
    if(this == obj){
        return true;
    }

    //为了提供程序的健壮性
    //我先判断一下，obj是不是学生的一个对象，如果是，再做向下转型，如果不是，直接返回false。
    //这个时候，我们要判断的是对象是否是某个类的对象?
    //记住一个格式：对象名 instanceof 类名
    //表示：判断该对象名是否是该类名一个对象
    if(!(obj instanceof Student)){
        return false;
    }
    //如果是就继续

    Student s = (Student)obj;
    // 比较每个属性的大小是否相同
    return this.name.equals(s.name) && this.age == s.age;

这个代码还有最终版本:

@Override
public boolean equals(Object obj) {
    if (this == obj)         // 对象相同
        return true;
    if (obj == null)         // 传入的参数为空
        return false;
    if (getClass() != obj.getClass())   // 编译的class文件不是同一个肯定不相同 等价于instanceOf
        return false;
    Student other = (Student) obj;     // 强制转成Student对象再进行比较
    if (age != other.age)              
        return false;
    if (name == null) {
        if (other.name != null)
            return false;
    } else if (!name.equals(other.name))
        return false;
    return true;
}

面试考法：==和equals()方法的区别？

A.==
基本类型：比较的是值是否相同
引用类型：比较的是地址值是否相同
B.equals()
只能比较引用类型。默认情况下，比较的是地址值是否相同。
但是，我们可以根据自己的需要重写该方法。
