自定义标签开发

发表于 2017-08-17 | 分类于 JavaWeb基础

编写一个实现Tag接口的Java类（标签处理类）

查看Jsp规范文档，发现Tag接口有五个方法，又恰好TagSupport类继承了Tag并且默认实现了它，于是我们选择直接继承Tag接口的已知实现类TagSupport，因为它实现了Tag接口所有方法，重写我们所需要的方法。

代码实现（以输出客户机IP为例子）

public class ViewIPTag extends TagSupport {

public int doStartTag() throws JspException {
   HttpServletRequest request=(HttpServletRequest) this.pageContext.getRequest();
   JspWriter out=this.pageContext.getOut();
   try {
        out.write(request.getRemoteAddr());
    } catch (IOException e) {
      throw new RuntimeException(e);
    }
    return super.doStartTag();
  }

}

配置标签库符(tld)文件

在编写tld文件可以参考*(\Tomcat_*\webapps\examples\WEB-INF\jsp2\jsp2-example-taglib) , 并且tld文件应该放在 WebRoot\WEB-INF** 目录下。

itcast.tld文件如下：

<?xml version="1.0" encoding="UTF-8" ?>
<taglib xmlns="http://java.sun.com/xml/ns/j2ee"
xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
xsi:schemaLocation="http://java.sun.com/xml/ns/j2ee http://java.sun.com/xml/ns/j2ee/web-jsptaglibrary_2_0.xsd"
version="2.0">
<!-- description用来添加对taglib(标签库)的描述 -->
<description>Xavier自定义标签库开发</description>
<tlib-version>1.0</tlib-version>
<short-name>SimpleTagLibrary</short-name>
<!-- 
 为自定义标签库设置一个uri,uri可以写成http://www.itcast.cn的形式便于像itcast.tld里面的所有标签都绑定到
 <uri>http://www.itcast.cn</uri>中。在jsp引用标签库的时候可以用<%@taglib uri="http://www.itcast.cn"%>的形式找到
 -->

<uri>http://itcast.cn</uri>

<!-- 
一个taglib(标签库)中包含多个自定义标签每一个自定义标签使用tag标记来描述
一个tag标签对应一个自定义标签
 -->
<tag>
<!-- 
为标签处理类配置一个标签名,在Jsp页面中使用标签时是通过标签名来找到要调用的标签处理器类,
通过ViewIP就能找到对应的标签处理的完整域名cn.itcast.web.tag.ViewIPTag
 -->
<name>ViewIP</name>
<tag-class>cn.itcast.web.tag.ViewIPTag</tag-class><!-- 映射到完整的class类上 -->
<body-content>empty</body-content><!-- 有无标签体的内容 -->
</tag>

jsp页面引用标签

<%@taglib uri="uri标签配置的内容" prefix="标签使用前缀"%>

自定义标签的执行流程

jsp引擎遇到自定义标签时，首先创建标签处理类的实例对象，然后按照jsp规范定义的通信规则一次调用它的方法。

1.public void setPageContext(PageContext pageContext) :

jsp引擎实例化标签处理器后，将调用setPageContext方法将jsp页面的pageContext对象传递给标签处理器，标签处理器以后可以通过这个pageContext对象与jsp页面进行通信。

2.public void setParent(Tag t):

setPageContext方法执行之后，web容器接着调用setParent方法将当前标签的父标签传给当前标签处理器，如果标签没有父标签，则传给setParent方法的参数值为null

3.public int doStartTag():

在调用完1，2两种方法之后，web容器执行自定义标签开始标记的时候开始调用doStartTag方法

4.public int doEndTag():

在web容器执行完自定义标签的标签体后，执行自定义标签结束标签时，就会调用标签处理器的doEndTag

5.public void release():

在web容器执行完自定义标签后，标签处理器会留在内存当中，为其他请求服务，直到web应用停止后，web容器才会调用release方法
