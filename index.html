Web容器

什么叫Web容器

Servlet没有main方法。Servlet受控于另一个Java应用，这个Java应用成为容器。Tomcat就是这样一
个容器。（而不是其他请求，如请求一个普通的静态如果web服务器一个用得到一个指向Servlet的请求
Html），此时服务器不是把这个请求交给servlet本身，而是交给部署该Servlet的容器，要由容器调用
Servlet方法，如doPost()或者doGet()方法。

容器能提供什么

通信支持：利用容器提供的方法，能让Servlet与Web服务器对话；

生命周期管理：容器控制着Servlet的生死。它会负责加载类，实例和初始化Servlet，调用Servlet方法，并使Servlet实例能够被垃圾回收。

声明方式实现安全：利用容器，可以使用Xml部署描述文件配置（或修改）安全性，而不必将硬编码写到Servlet（或其他）类代码中。

Jsp支持：负责将Jsp代码翻译成java文件放在 Tomcat\work\Catalina\localhost\web应用 下。

Web容器的工作原理

Web容器是一种服务程序，是能够支持发布web程序的软件，在服务器一个端口就有一个提供相应服务的程序，而这个程序就是处理从客户端发出的请求，如Java种的Tomcat容器，ASP的IIS或者PWS都是这样的容器。一个服务器可以有多个容器。

在B/S结构下，以常见的自来水供水系统而言。整个Web系统好比完整的供水系统，用户只需要有水龙头(Browser)，水管相当于网络（Network），自来水公司的供水系统（Server），其中的压力水罐就是容器（Container）。

浅析Tomcat

Tomcat的简单介绍

Tomcat是一个小型的轻量级应用服务器，很瘦广大程序员的喜欢。因为它它运行时占用的系统资源小，扩展性好支持负载平衡与邮件服务等开发应用系统常用的功能；而且它还在不断的改进和完善中。

一般认为，当在一台机器上配置好Apache服务器，可利用它响应对Html页面和Servlet。实际上Tomcat部分是Apache服务器的扩展，但它是独立运行的，所以当你运行Tomcat时，它实际上作为一个与Apache独立的进程单独运行。

当配置正确时，Apache为Html页面服务，而Tomcat实际上运行Jsp页面和Servlet。另外，Tomcat和IIS，Apache等Web服务器一样，具有处理Html页面的功能，另外它还是一个Servlet和Jsp容器，独立的Servlet容器是Tomcat的默认模式。不过，Tomcat处理静态Html的能力不如Apache服务器。

Tomcat是Apache-Jarkarta的一个子项目，是一个开放式源码，免费支持Jsp和Servlet技术的容器，它同时又是一个Web服务器软件。Servlet和Jsp两者分工协作，Servlet侧重于解决运算和业务逻辑问题，Jsp则重点解决展示问题。

Tomcat目录结构分析

Web应用程序需要部署到Web容器种才能运行，两者都必须符合JavaEE规范。Web容器的应用程序规范如下：

1.目录结构规范：JavaWeb程序的所有文件都包含在一个文件夹中，包含WEB-INF子文件夹和META-INF子文件夹，WEB-INF文件夹包含classes文件夹和lib文件夹以及web.xml。

Tomcat的结构目录如下：

/bin：包含启动和关闭Tomcat的文件

/conf：包含不同的配置文件:server.xml,web.xml,tomcat-user.xml

/lib：包含Tomcat使用的Jar文件

/logs：包含日志文件

/webapps：包含应用程序实例及自己开发的程序

/work：包含Jsp生成的Servlet（即.java文件）

2.Jar文件放在WEB-INF/lib文件夹中。

3.web.xml规范：是整个web应用程序的描述文件，里面定义者该Web应用程序可供访问的信息资源以及这些资源的行为，包括Servlet，过滤器，监听器，安全验证等信息资源。

4.其他资源文件，如jsp，html，图片和声音的放在与WEB-INF同一级目录下，因为WEB-INF文件夹对于客户端是不可见的。

自定义标签的执行流程

jsp引擎遇到自定义标签时，首先创建标签处理类的实例对象，然后按照jsp规范定义的通信规则一次调用它的方法。

1.public void setPageContext(PageContext pageContext) :

jsp引擎实例化标签处理器后，将调用setPageContext方法将jsp页面的pageContext对象传递给标签处理器，标签处理器以后可以通过这个pageContext对象与jsp页面进行通信。

2.public void setParent(Tag t):

setPageContext方法执行之后，web容器接着调用setParent方法将当前标签的父标签传给当前标签处理器，如果标签没有父标签，则传给setParent方法的参数值为null

3.public int doStartTag():

在调用完1，2两种方法之后，web容器执行自定义标签开始标记的时候开始调用doStartTag方法

4.public int doEndTag():

在web容器执行完自定义标签的标签体后，执行自定义标签结束标签时，就会调用标签处理器的doEndTag

5.public void release():

在web容器执行完自定义标签后，标签处理器会留在内存当中，为其他请求服务，直到web应用停止后，web容器才会调用release方法

Lintcode学习笔记之二叉树后序遍历

发表于 2017-08-07 | 分类于 算法学习

题目描述

Given a binary tree, return the postorder traversal of its nodes’ values.

例子

return [3,2,1].

思考

  1. 本质上和前后序遍历没有什么区别。
  2. 增加一个pre结点作为标志，标记之前的结点的位置，为了表示右孩子到底有没有遍历完从而决定要不要从栈当中弹出根节点。此外其实都是之前的老套路，还是先找到叶子结点的最左侧。

代码实现(不展示递归方法)

 // Iteration :
下面在弹栈的时候需要分情况一下：
1）如果当前栈顶元素的右结点存在并且还没访问过（也就是右结点不等于上一个访问结点），
那么就把当前结点移到右结点继续循环；
2）如果栈顶元素右结点是空或者已经访问过，那么说明栈顶元素的左右子树都访问完毕，
应该访问自己继续回溯了。
下面列举一下代码：
     public class Solution{
 public ArrayList<Integer> postorderTraversal(TreeNode root) {  
 ArrayList<Integer> postOrder=new ArrayList<Integer>(); 
 Stack<TreeNode> stack=new Stack<TreeNode>(); 
 TreeNode pre=null,curr=root; //初始化当前的结点为root而pre(之前结点)为null

while(curr!=null||!stack.empty()){  //遍历结束的条件是栈的值为空或者当前结点为空
 if(curr!=null){            // 为了找到叶子结点的最左侧
   stack.push(curr);
   curr=curr.left;
 }else{                     // 已经找到最左端的叶子结点
   TreeNode peekNode=stack.peek();  // 不确定其还有没有右孩子所有不用pop方法
   if(peekNode.right!=null&&pre!=peekNode.right){ // 如果该结点有右孩子并且它还没有被遍历过
   curr=peekNode.right;  //将其右孩子替换成当前结点
   }else{   //有两种情况 1. 其没有右孩子 2.其有右孩子 但是已经被遍历过了
   stack.pop();     // 弹出该结点 
   postOrder.add(peekNode.val);    
   pre=peekNode;   // 修改pre(之前结点) ，将该结点作为当前结点 
 }
    }
  }
return postOrder;
    }
 }


//version2 :Iteration
 思路：
（1）如果pre的左孩子或者右孩子是cur，那么说明遍历在往下走，按访问顺序继续，
 即如果有左孩子，则是左孩子进栈，否则如果有右孩子，则是右孩子进栈，
 如果左右孩子都没有，则说明该结点是叶子，可以直接访问并把结点出栈了。
 (2）如果反过来，cur的左孩子是pre，则说明已经在回溯往上走了，
 但是我们知道后序遍历要左右孩子走完才可以访问自己，
 所以这里如果有右孩子还需要把右孩子进栈，否则说明已经到自己了，可以访问并且出栈了。
（3）如果cur的右孩子是pre，那么说明左右孩子都访问结束了，可以轮到自己了，
 访问并且出栈即可。 
 public class Solution{
  public ArrayList<Integer> postorderTraversal(TreeNode root) {  
  ArrayList<Integer> res = new ArrayList<Integer>();  
  if(root == null){ return res;}  
  Stack<TreeNode> stack = new Stack<TreeNode>();  
  stack.push(root);  
  TreeNode pre = null;  
  while(!stack.Empty()) {  
  TreeNode cur = stack.peek(); //找到栈顶的值  
  if(pre==null || pre.left==cur || pre.right==cur){ 
  //1.为null还没开始开始遍历  2，3 表示 二叉树在往下遍历的过程中   
        if(cur.left!=null) {    
            stack.push(cur.left);  
        } else if(cur.right!=null){  
            stack.push(cur.right);  
        }else{               // 没有左右孩子说明已经是在叶子结点，可以弹出栈
            cur=stack.pop();
            res.add(cur.val);  
        }  
    }else if(cur.left==pre && cur.right!=null){  
   // 说明二叉树在回溯的过程中并且当前右孩子结点还没有遍历过
        stack.push(cur.right);  
    }else{ // 1.cur没有右孩子  2.cur有右孩子结点但是已经遍历完毕  
        res.add(cur.val);  
        stack.pop();  
    }  
    pre = cur;    //修改当前pre(之前结点)的值 
}  
return res;  
 } 
}

问题描述

Given a binary tree, return the inorder traversal of its nodes’ values.

例子

Given binary tree {1,#,2,3},

return [1,3,2].

思考

  1. 非递归方法和先序遍历相差无几，还是需要找到叶子结点的最左端。
  2. 还是可以用分治法来解决

// version1 : 递归(recrusion) 
public class Solution{
    ArrayList<Integer> inorder=new ArrayList<Integer>();
    public ArrayList<Integer> inorderTraversal(TreeNode root) {
    // write your code here
    if(root==null){ return inorder;}

    inorderTraversal(root.left);
    inorder.add(root.val);
    inorderTraversal(root.right);
    return inorder;
   }
}

左孩子结点为空时的情况如下：

//version2 :  非递归（迭代）
 public class Solution{
 public ArrayList<Integer> inorderTraversal(TreeNode root) {
 ArrayList<Integer> res=new ArrayList<Integer>();
 Stack<TreeNode> stack=new Stack<TreeNode>();
 while(root!=null||!stack.empty()){
    if(root!=null){
        stack.push(root);
        root=root.left;
    }else{
        root=stack.pop();
        res.add(root.val);
        root=root.right;
    }
}
    return res;
  }

//version3 :  分治法(Divide & Conquer)
public class Solution{

 public ArrayList<Integer> inorderTraversal(TreeNode root){
     ArrayList<Integer> inorder=new ArrayList<Integer>(); //inorder的值要放在方法体里面
     if(root==null){ return inorder;}
     //Divide 
     ArrayList<Integer> left=inorderTraversal(root.left);
     ArrayList<Integer> right=inorderTraversal(root.right);

     //Conquer
     inorder.addAll(left);
     inorder.add(root.val);
     inorder.addAll(right);
     return inorder; 
 }
}

Lintcode学习笔记之Balanced Binary Tree

发表于 2017-07-31 | 分类于 算法学习

问题描述

Given a binary tree, determine if it is height-balanced.
For this problem, a height-balanced binary tree is defined as a binary tree in which the depth of the two subtrees of every node never differ by more than 1.

例子

Given binary tree A = {3,9,20,#,#,15,7}, B = {3,#,20,15,7}

思考

分支条件
1.空节点的二叉树
2.在根节点的下的左右子树的深度相差已经>1
3.在2条件不成立，继续遍历分别以左右子树作为根节点的情况

代码展示

/**
 * //定义一棵树 
 * Definition of TreeNode:
 * public class TreeNode {
 * public int val;
 * public TreeNode left, right;
 * public TreeNode(int val) {
 * this.val = val;
 * this.left = this.right = null;
 * }
 * }
 */

public class Solution {
/**
 * @param root: The root of binary tree.
 * @return: True if this Binary tree is Balanced, or false.
 */
 // TotalTime :1947ms
public boolean isBalanced(TreeNode root) {
// write your code here
if(root==null) {return true;} // 根节点为空的情况返回 true
int left=depth(root.left);    // 通过depth(TreeNode)方法得到左子树的深度
int right=depth(root.right);  // 同理
if(left-right>1||left-right<-1){ //两颗子树的深度相差超过1
return false;
}else{
return isBalanced(root.left)&&isBalanced(root.right); // 继续与左右子树作为根节点进行遍历
}

}
private int depth(TreeNode node){
if(node==null){
return 0;
}else if(node.left==null&&node.right==null){
return 1;
}else{
int left=depth(node.left);
int right=depth(node.right);
return 1+(left>right?left:right);
}
}
}

例子

Given 7->1->6 + 5->9->2. That is, 617 + 295.

Return 2->1->9. That is 912.

Given 3->1->5 and 5->9->2, return 8->0->8.

思考

1.如果两个数的位数不同怎么办？
答：设置两个分支，分别是两个数在特定的位上都有数和特定的位置只有其中一个数（那个更长的）的情况。
2.怎么处理位数溢出的要求？
答：给定一个变量carry记录变化，把它放在链表的最后一位。
3.怎样设置链表能显示计算结果是倒置的效果 ？
答：按照链表的顺序相加即可。

代码实现

/**
 * Definition for singly-linked list.
 * public class ListNode {
 * int val;
 * ListNode next;
 * ListNode(int x) {
 * val = x;
 * next = null;  
 * }
 * }
 */

public class Solution {

/**
 * @param l1: the first list
 * @param l2: the second list
 * @return: the sum list of l1 and l2 
 */
public ListNode addLists(ListNode l1, ListNode l2) {
// write your code here
if(l1==null&&l2==null){ //判断链表是否为空
return null;
}

ListNode head=new ListNode(0);//设置一个空的链表头，这是一个套路，删除链表的时候也会用到，用以避免处理表头是否需要删除的问题。
ListNode point=head;  //用point作为具体操作的结点
int carry=0;          //定义进位数字标志位
while(l1!=null&&l2!=null){
int sum=l1.val+l2.val+carry;
point.next=new ListNode(sum%10);  //保留个位
carry=sum/10;                     //保留个位之外的数字
l1=l1.next;
l2=l2.next;
point=point.next;
}

while(l1!=null){                 //如果l1是那个更长的链表则用l1进行相加
int sum=l1.val+carry;
point.next=new ListNode(sum%10);
carry=sum/10;
l1=l1.next;
point=point.next; 
}
while(l2!=null){                //如果l2是那个更长的链表则用l1进行相加
int sum=l2.val+carry;
point.next=new ListNode(sum%10);
carry=sum/10;
l2=l2.next;
point=point.next; 
}

if(carry!=0){                  //进位数标志位不为0时说明要再设置一位，数值就为carry的值
point.next=new ListNode(carry);
}
return head.next;
}
}
